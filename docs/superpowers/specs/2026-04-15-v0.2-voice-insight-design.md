# Spoke v0.2 — Voice to Insight

**Date:** 2026-04-15
**Status:** Approved for implementation

---

## Problem

The current app translates voice recordings into engagement-optimized formats: 5 tweet angles, a thread, long-form, an Instagram caption, with an optional framework overlay. The output is oriented around structure and reach.

The actual need is different. Speaking is the most natural form of communication for this user, but the goal is writing — specifically writing that makes the reader look at themselves, reflect, or arrive at an insight they hadn't named before. Engagement metrics, writing frameworks, and format optimization are not the goal. The writing itself is.

---

## What Changes

### New intent

Spoke is a voice-to-insight translator. Record something you're thinking about. Spoke finds the truth in it and writes it in your voice — as a piece that makes a reader recognize something about themselves.

### What gets removed

- Tweet options (5 angles)
- Thread format
- Long-form tab
- Instagram caption
- Framework overlay ("Apply a framework" button)
- JK Molina preset
- `api/framework.js` endpoint

### What gets added

- `SetupScreen` — one-time onboarding (writing intent + voice examples)
- New Claude prompt in `api/recordings.js`
- Rebuilt `OutputScreen` — essay + moments, single page

---

## Screen Flow

```
First visit:    SetupScreen → RecordScreen → LoadingScreen → OutputScreen
Return visits:  RecordScreen → LoadingScreen → OutputScreen
```

SetupScreen is shown when localStorage has no voice profile saved. A settings icon on RecordScreen provides access to edit the profile at any time.

---

## SetupScreen

Two steps, shown once on first visit.

### Step 1 — Writing intent

The user picks what they want their writing to do. Displayed as selectable tiles:

- **Insight / Reflection** — writing that makes the reader see something about themselves
- **Education** — writing that teaches something clearly and simply
- **Storytelling** — writing that pulls the reader into an experience
- **Motivation** — writing that moves someone to act
- **Custom** — opens a text field where the user describes their intent in their own words

One selection required before proceeding. The selected intent (either the preset label or the custom text) is stored in localStorage under `spoke_writing_intent`.

### Step 2 — Voice examples

A large text area with the label: "Paste a few things you've written that sound like you."

A note below: "3–5 pieces is enough. The more specific to your voice, the better."

A "Save and start" button stores the text in localStorage under `spoke_voice_examples` and navigates to RecordScreen.

No processing or parsing of the examples — raw text stored and passed directly to Claude.

Unchanged except:

- Settings icon (top right corner) opens a modal with the SetupScreen content pre-filled — allows editing writing intent and voice examples at any time.

## API — `api/recordings.js`

### Request

Unchanged: `transcript` (pre-built) or `audio_base64` + `audio_type`.

New fields (sent by client):

- `voice_examples` — string, the user's saved writing examples
- `writing_intent` — string, the user's writing intent (preset label or custom text)

`RecordScreen` reads both values from localStorage and passes them to `submitRecording(audioBlob, { voiceExamples, writingIntent })`. `api.js` includes them in the POST body. If either is missing (e.g. cleared storage), the API falls back gracefully: the prompt omits the missing sections and Claude writes without voice reference or specific intent.

### Claude prompt (new)

The prompt is rebuilt entirely. The structure:

```
You are a writing collaborator helping [author] turn spoken thoughts into insight-driven writing.

The goal: find the truth in this transcript and write it as a piece that makes a reader 
recognize something about themselves — without engagement tactics, hooks, or writing frameworks. 
The insight earns the reader's attention.

The author's writing intent: {WRITING_INTENT}

Here are examples of the author's writing. Match their sentence length, rhythm, specificity, 
and cadence. Do not copy these — write in the same register:

{VOICE_EXAMPLES}

Raw transcript:
{TRANSCRIPT}

Output ONLY this exact JSON structure — no markdown, no explanation:
{
  "essay": "<500-800 word essay>",
  "moments": ["<variation 1>", "<variation 2>", "<variation 3>"]
}

Essay rules:
- 500-800 words
- Structured around the core insight in the transcript
- Opens with something specific and grounded — not abstract
- The reader should recognize themselves somewhere in it
- Sounds like the author wrote it after thinking hard about something, not like AI
- No throat-clearing, no "In this piece I will..." setup
- No corporate or AI-sounding language

Moment rules:
- Exactly 3 variations
- 150-250 words each
- Same insight as the essay, compressed
- Each one is a complete, standalone piece — not a summary or excerpt
- Different compressions of the same truth, not different angles or hooks
- Match the author's voice as closely as the essay does
- Short sentences, fragments, and line breaks are fine — preserve the spoken rhythm

Do not add ideas that weren't in the transcript.
Do not impose a structure that wasn't in the original thinking.
Do not optimize for engagement, likes, or shares.
```

### Response

```json
{
  "transcript": "...",
  "formats": {
    "essay": "...",
    "moments": ["...", "...", "..."]
  }
}
```

The `formats` key is preserved for backwards compatibility with the App.jsx state shape. `tweet`, `thread`, `longform`, and `caption` are no longer returned.

## OutputScreen

Single page, stacked. No tabs.

### Essay section

- Full essay text, readable
- Copy button below

### Moments section

- Header: "Moments"
- 3 cards stacked vertically
- Each card: the moment text + a copy button
- Label above each: "Option 1", "Option 2", "Option 3"

### No framework section

The "Apply a framework" button and the framework panel are removed entirely.

### New recording

"+ New recording" button at the bottom, unchanged.

## localStorage Keys

| Key | Value | Set by |
|-----|-------|--------|
| `spoke_writing_intent` | string (preset label or custom text) | SetupScreen / Settings |
| `spoke_voice_examples` | string (raw pasted text) | SetupScreen / Settings |

If either key is missing, SetupScreen is shown.

## What Stays Unchanged

- Audio recording logic (`RecordScreen`, `audioChunker.js`)
- Whisper transcription (`api/transcribe.js`)
- Chunking threshold (2.5MB)
- Vercel Functions structure
- `CopyButton` component
- `LoadingScreen`
- `api.js` fetch wrappers (`submitRecording` updated to accept and forward `voiceExamples` and `writingIntent`)
